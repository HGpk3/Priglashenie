# Project Context

Обновлено: 2026-04-13

## О проекте

Мы делаем сайт-электронное приглашение на свадьбу, которая состоится летом.

Сайт должен решать две задачи:

1. Эмоционально передать атмосферу события.
2. Дать гостям всю практическую информацию в удобном формате.

## Текущий статус

- Репозиторий инициализирован.
- PDF-макет добавлен в проект: `design/wedding-invitation-mockup.pdf`
- Из PDF извлечены визуальные ассеты в `design/extracted/`
- Кодовая база ещё не начата.
- На старте зафиксированы правила, план, структура контента и список открытых вопросов.

## Что уже известно

- Формат: сайт-приглашение.
- Событие: свадьба.
- Период: лето.
- В основе визуальной части лежит PDF-макет, экспортированный из Figma.

## Что подтверждено по макету

- Дата события: `13.08.2026`
- Локация: `Four Seasons Lion Palace`
- В макете заложен длинный вертикальный лендинг, а не набор отдельных страниц.
- Внутри структуры уже есть блоки с таймингом, дресс-кодом, RSVP, пожеланиями и обратной связью.
- Предусмотрен счётчик до даты свадьбы.

## Предварительно распознанные блоки

- Первый экран с парой и пригласительным сообщением
- Эмоциональный текст-приглашение
- Дата и место проведения
- Тайминг дня
- Дресс-код с палитрой
- RSVP-анкета
- Пожелания по подаркам и цветам
- Блок с вопросами / контактами
- Счётчик до события

Часть формулировок восстановлена по PDF автоматически и потребует ручной сверки.

## Предварительные продуктовые ожидания

- Сайт должен хорошо работать на телефоне.
- Интерфейс должен быть лёгким, понятным и эмоциональным.
- Контент должен быть собран в короткие, легко сканируемые блоки.
- Ключевая информация должна находиться быстро без лишних кликов.

## Визуальное направление по PDF

- Настроение: тёмное, редакционное, атмосферное, ближе к fashion-съёмке, чем к классическому пастельному свадебному сайту
- Базовые образы: чёрный фон, глубокий бордовый, серебристый блеск, тёплый свет, контрастные фотографии пары
- Декоративные мотивы: розы, блестящая ткань, отпечаток-подобное сердце, вырезанные предметные изображения
- Формат композиции: вертикальный storytelling с выразительными визуальными акцентами

## Источники истины

Приоритет источников для принятия решений:

1. Подтверждённый PDF-макет
2. Согласованные тексты и данные от пары
3. Зафиксированные правила проекта
4. Технические решения внутри репозитория

## Рабочие допущения

- Сначала проектируем мобильный сценарий, затем расширяем на десктоп.
- На первом этапе сайт можно делать как одностраничный.
- Все неподтверждённые данные храним как плейсхолдеры и явно помечаем.
- Извлечённые изображения из `design/extracted/` используем как референс и вспомогательный материал, но главным исходником остаётся PDF.

## Ограничения

- Пока не выбраны стек, структура данных и способ RSVP.
- Пока не подтверждены все тексты, точный адрес, контакты и финальные формулировки пожеланий.
- Часть текста в PDF извлекается с искажениями, поэтому нужна ручная сверка смысловых блоков.

## Следующее действие

На следующем шаге:

1. Закрыть открытые вопросы по контенту.
2. Выбрать стек.
3. Начать реализацию лендинга по структуре из `docs/CONTENT_STRUCTURE.md`.
