# Project Context

Обновлено: 2026-05-05

## О проекте

Мы делаем сайт-электронное приглашение на свадьбу, которая состоится летом.

Сайт должен решать две задачи:

1. Эмоционально передать атмосферу события.
2. Дать гостям всю практическую информацию в удобном формате.

- Интерфейс должен быть лёгким, понятным и эмоциональным.
- Контент должен быть собран в короткие, легко сканируемые блоки.
- Ключевая информация должна находиться быстро без лишних кликов.

## Визуальное направление по PDF

- Настроение: тёмное, редакционное, атмосферное, ближе к fashion-съёмке, чем к классическому пастельному свадебному сайту
- Базовые образы: чёрный фон, глубокий бордовый, серебристый блеск, тёплый свет, контрастные фотографии пары
- Декоративные мотивы: розы, блестящая ткань, отпечаток-подобное сердце, вырезанные предметные изображения
- Формат композиции: вертикальный storytelling с выразительными визуальными акцентами

## Текущее дизайн-решение по первому экрану

- Первый экран строится как кинематографичный `Editorial Cover`: фото пары, крупные русские имена, дата, место и декоративные слои.
- Главный hero-ассет: `design/extracted/obj-163_2332x3498.jpg`.
- Буквальное повторение PDF на hero больше не является целью; приоритет у сильного веб-исполнения, которое сохраняет настроение макета.
- Анимации первого экрана: проявление фото и текста при загрузке, медленное дыхание изображения, float декоративных элементов, shimmer даты и scroll-cue.
- После первичной оценки hero усилена видимость фотографии пары: уменьшены затемняющие слои и декоративные фактуры, дата вынесена из нижней зоны кадра.
- Hero-фото больше не используется как full-screen фон: из исходного `obj-163_2332x3498.jpg` подготовлен отдельный crop `design/extracted/hero-couple-crop.jpg`, чтобы фото-объект держал пару в фокусе без обрезанных лиц.
- Имена пары в hero выводятся строгими русскими капителями Cormorant Garamond; casual/script-вариант для hero отклонён.
- Отдельный date-badge в hero заменён на типографическую date-line под именами, чтобы первый экран выглядел цельнее.
- Для первого появления добавлена короткая свадебная прелюдия с эффектом письма пером: прорисовываются `Danil & Elena` и дата, затем открывается основная hero-композиция с русскими именами.
- Фон прелюдии усилен по яркости и винному оттенку, чтобы “штора” была заметнее перед раскрытием hero.
- Для мобильного сценария hero адаптирован отдельно под формат iPhone 14 Pro Max: фото больше не выглядит маленькой нижней вставкой, а становится центральным визуальным объектом под именами и датой.

## Текущее дизайн-решение по RSVP

- Анкета оформляется как `RSVP-открытка`, а не как обычная веб-форма.
- Deadline `до 20 июня 2026` подаётся как отдельный штамп.
- Вопросы остаются на одном экране, чтобы гостям было удобно заполнить всё без лишних шагов.
- Варианты выбора в RSVP оформлены прямоугольными плитками, а не круглыми чипами.
- После успешной отправки появляется рукописная подпись `Спасибо`.
- RSVP подключается через `POST /api/rsvp`: сервер валидирует обязательные поля, проверяет допустимые варианты, ограничивает частые отправки, отсекает простые bot-submission через honeypot и отправляет сообщение в Telegram Bot API.
- Для рабочей отправки в Telegram в `.env` должны быть заполнены `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID`; `TELEGRAM_THREAD_ID` нужен только для топиков в группе.

## Рабочие допущения

- Сначала проектируем мобильный сценарий, затем расширяем на десктоп.
- На первом этапе сайт можно делать как одностраничный.
- Все неподтверждённые данные храним как плейсхолдеры и явно помечаем.
- Извлечённые изображения из `design/extracted/` используем как референс и вспомогательный материал, но главным исходником остаётся PDF.

## Ограничения

- Локальная проверка выполняется через Docker, основной сценарий запуска: `docker compose up -d --build`.
- Пока не подтверждены все тексты, точный адрес, контакты и финальные формулировки пожеланий.
- Часть текста в PDF извлекается с искажениями, поэтому нужна ручная сверка смысловых блоков.
