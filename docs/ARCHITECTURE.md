# Architecture

Обновлено: 2026-05-15

## Стек

- `Next.js 15`
- `React 19`
- `TypeScript`
- `App Router`
- `Docker Compose`
- `Nginx` как reverse proxy

## Слои проекта

- `app/` - маршруты, layout, глобальные стили и API.
- `components/` - презентационные и интерактивные секции лендинга.
- `data/` - контент приглашения, дизайн-токены и настройки, которые не должны жить внутри JSX.
- `fonts/` - локальные шрифты, подключаемые через `next/font/local`.
- `public/` - публичные ассеты.
- `infra/` - деплойная инфраструктура.
- `docs/` - проектная документация.

## Runtime-схема

В продакшене приложение запускается через `docker-compose.yml`:

- `app` собирается из `Dockerfile` как Next.js standalone build.
- `proxy` использует `nginx:1.27-alpine` и прокидывает внешний HTTP-трафик в `app:3000`.
- `rsvp-data` хранит файл RSVP-ответов вне жизненного цикла контейнера.

Внешний порт задается через `HTTP_PORT`. Домен задается через `DOMAIN_NAME`; для текущего проекта используется `24wedding.ru www.24wedding.ru`.

## RSVP

Форма отправляет данные на `POST /api/rsvp`. Сервер:

- валидирует обязательные поля;
- отсекает honeypot-поле;
- ограничивает частые отправки;
- сохраняет ответы в JSONL-файл;
- отдает результаты через `GET /api/rsvp?key=...`.

Путь хранения внутри контейнера: `/app/.data/rsvp-submissions.jsonl`.

## Healthcheck

`GET /api/health` используется Docker healthcheck-ом и может быть полезен для внешнего мониторинга.
