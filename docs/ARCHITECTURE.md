# Architecture

Обновлено: 2026-04-13

## Выбранный стек

- `Next.js`
- `App Router`
- `TypeScript`
- глобальные стили + модульные стили секций

## Почему выбираем Next.js

- Пользователю удобнее dev-режим и общая экосистема
- Проект можно начать как простой лендинг и при необходимости безболезненно дорастить до форм, API-роутов и более сложной логики
- Структура `app/` хорошо подходит для секционного сайта с понятной композицией

## Стартовая структура проекта

- `app/layout.tsx` - базовый layout
- `app/page.tsx` - стартовая страница приглашения
- `app/page.module.css` - визуальная система стартового лендинга
- `app/globals.css` - базовые глобальные стили
- `data/site-content.ts` - контент и подтверждённые данные отдельно от вёрстки
- `docs/` - проектные решения, правила и открытые вопросы
- `design/` - PDF-макет и извлечённые референсы

## Архитектурные принципы

- Контент отделяем от интерфейса
- Секции страницы проектируем как независимые логические блоки
- Подтверждённые данные не смешиваем с временными плейсхолдерами
- RSVP реализован как клиентская форма `components/landing/RsvpFormSection.tsx` и серверный endpoint `app/api/rsvp/route.ts`, который валидирует payload и отправляет ответы в Telegram Bot API.
- Визуальная выразительность не должна ломать мобильный опыт
