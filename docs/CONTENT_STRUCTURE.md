# Content Structure

Обновлено: 2026-04-13

Этот файл собирает структуру лендинга по PDF-макету.

Где данные подтверждены точно, это указано явно. Где текст был восстановлен автоматически из PDF, это помечено как `предварительно`.

## 1. Hero / Первый экран

Статус: `предварительно`

- Крупный первый экран с фотографией пары
- Пригласительный заголовок
- Имена пары
- Дата свадьбы

## 2. Приветственный текст

Статус: `предварительно`

- Эмоциональный абзац с приглашением гостей на свадьбу
- По смыслу это тёплое личное сообщение от пары

## 3. Основные детали

Статус: `частично подтверждено`

- Дата: `13.08.2026`
- Место проведения: `Four Seasons Lion Palace`
- Вероятно есть отдельные подписи вроде `дата свадьбы` и `место проведения`

## 4. Тайминг дня

Статус: `предварительно`

- `14:45` - сбор гостей у Петроградского ЗАГСа
- `15:00` - торжественная церемония
- `16:00` - трансфер до ресторана
- `17:00` - праздничный банкет

Нужна ручная сверка этих формулировок.

## 5. Dress Code

Статус: `частично подтверждено`

Из PDF удалось распознать палитру, близкую к такой:

- тёмный изумруд
- тёплый графит
- пыльная лаванда
- серо-голубой
- глубокий оливковый
- пепельная роза
- пудра
- тёмно-синий
- чёрный
- бордовый
- шоколад
- серебро

Также в макете есть визуальные примеры образов.

## 6. RSVP / Анкета

Статус: `частично подтверждено`

По структуре макета в анкете, вероятно, есть такие поля:

- будете ли присутствовать
- имя и фамилия
- какой алкоголь предпочитаете
- предпочитаете рыбу или мясо
- есть ли аллергия на что-либо
- требуется ли трансфер до ресторана
- кнопка отправки

Также по распознаванию видны варианты ответов для напитков и еды.

## 7. Пожелания

Статус: `предварительно`

По смыслу макет содержит отдельный блок с организационными пожеланиями:

- предпочтение денежного подарка
- просьба не дарить цветы
- возможная альтернатива в виде бутылки алкоголя
- комментарий про курение и отдельное место

Точные формулировки нужно подтвердить.

## 8. Countdown

Статус: `подтверждено`

- В макете есть счётчик до свадьбы
- Используются подписи по времени: дни, часы, минуты, секунды

## 9. Вопросы / Контакты

Статус: `предварительно`

- Есть отдельный блок на случай, если у гостей останутся вопросы
- Вероятно там должны быть контакты пары или ответственных людей

## Что это значит для реализации

- По структуре это одностраничный длинный лендинг
- Контент нужно хранить отдельно от верстки
- RSVP лучше проектировать как самостоятельный блок с гибким способом отправки
- Дресс-код и пожелания уже являются обязательными разделами, а не второстепенными
